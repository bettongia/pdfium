# v0.03 ✅ Complete

## Page rendering to pixel buffer ✅ Complete

Render PDF pages to raw BGRA pixel data using `FPDF_RenderPageBitmap()` on the
PDFium isolate. The output is a platform-agnostic `Uint8List` that callers can
decode into an image using whichever graphics layer they have available.

**Delivered:**

- `PdfDocument.renderPageToBytes(pageIndex, pixelWidth, pixelHeight, {...})` —
  renders a page at the requested pixel dimensions and returns
  `({Uint8List pixels, int pixelWidth, int pixelHeight})` (BGRA format).
- `PdfDocument.getPageSize(pageIndex)` → `PdfPageSize` — intrinsic page
  dimensions in PDF points (1/72 inch).
- `PdfPageSize.sizeForDpi(dpi)` → `({double width, double height})` — converts
  from points to pixel dimensions at a given DPI.

Rendering flags exposed on `renderPageToBytes`:

| Parameter           | PDFium flag     | Default      |
| ------------------- | --------------- | ------------ |
| `renderAnnotations` | `FPDF_ANNOT`    | `true`       |
| `lcdText`           | `FPDF_LCD_TEXT` | `false`      |
| `backgroundColor`   | ARGB packed int | `0xFFFFFFFF` |

**Architectural constraint:** all PDFium calls run on the `PdfiumIsolate`
singleton — rendering is no exception. `renderPageToBytes` dispatches to the
isolate and awaits the result; the caller's isolate is never blocked.

**Coordinate system:** PDF origin is bottom-left; screen origin is top-left. Use
`FPDF_PageToDevice()` / `FPDF_DeviceToPage()` for all coordinate conversions.
