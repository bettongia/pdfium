# v0.03.01 ✅ Complete

## Table of contents extraction ✅ Complete

Plan: [TOC](../plans/completed/plan_toc.md)

**Complete.** `PdfDocument.tableOfContents` returns the full bookmark/outline
tree as `Future<List<PdfTocEntry>>`. `bin/pdfinfo.dart` gained a `--toc` flag
for plain-text and JSON output.

## Image extraction ✅ Complete

Plan: [Image Extraction](../plans/completed/plan_image_extraction.md)

**Complete.**
`PdfDocument.extractImages({int? pageIndex, bool includeBitmap = false})`
streams `PdfPageImages` per page, and
`PdfDocument.renderImage(pageIndex, objectIndex)` fetches `PdfImageBitmap` on
demand. New types: `PdfColorspace`, `PdfImageMetadata`, `PdfImage`,
`PdfImageBitmap`, `PdfPageImages`.

## Thumbnail extraction ✅ Complete

Plan: [Thumbnail Extraction](../plans/completed/plan_thumbnail_extraction.md)

**Complete.**
`PdfDocument.getThumbnail(pageIndex, {generateIfAbsent, maxDimension})` returns
a `PdfThumbnail` with BGRA pixel data. Embedded `/Thumb` streams are decoded via
`FPDFPage_GetThumbnailAsBitmap` (`fpdf_thumbnail.h`). When no embedded thumbnail
is present and `generateIfAbsent` is `true` (the default), the page is rendered
at `maxDimension` pixels on the longest edge. New types: `PdfThumbnailSource`,
`PdfThumbnail`.

## Search ✅ Complete

Plan: [Search](../plans/completed/plan_search.md)

**Complete.** `PdfDocument.search(query, {flags, pageIndex})` streams
`PdfSearchMatch` results page-by-page. New types: `PdfSearchFlag`,
`PdfSearchMatch`. `bin/pdfinfo.dart` gained a `--search <query>` flag for
plain-text and JSON output. Bounding rectangles are in PDF user space (origin
bottom-left, units in points), consistent with `PdfRect` throughout the library.
