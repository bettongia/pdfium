# v0.02 ✅ Complete

## Text extraction ✅ Complete

> Implemented in
> [`../plans/completed/plan_text_extraction.md`](../plans/completed/plan_text_extraction.md).
> Layout-aware reordering (multi-column, RTL) is deferred to v0.06.

Extract Unicode text per page using `FPDFText_LoadPage()` /
`FPDFText_GetText()`. Also expose bounded-text extraction, character-level
bounding boxes, built-in search (`FPDFText_FindStart/Next/Prev`), and
per-character font metadata (name, size, weight, colour) for heading/structure
detection.

**Known edge cases that must be covered in tests:**

- **Scanned PDFs** have no text layer — PDFium returns nothing; OCR is out of
  scope.
- **Multi-column and RTL text** — extraction order may not match reading order;
  layout-aware heuristic reordering is handled in the separate plan above.
- **Unmapped characters** — `FPDFText_GetUnicode()` returns 0;
  `FPDFText_GetText()` silently skips them. Use `FPDFText_HasUnicodeMapError()`
  to signal the gap.

**Architectural constraint:** All PDFium calls must run on a dedicated `Isolate`
with message-passing — PDFium is not thread-safe and must never be called from
the caller's isolate directly.

## Annotation extraction ✅ Complete

**Complete** (2026-05-20).

> Plan: `docs/plans/completed/plan_annotation_extraction.md`

Read all annotation types using `fpdf_annot.h`:

| Type                 | Constant                                       |
| -------------------- | ---------------------------------------------- |
| Sticky notes         | `FPDF_ANNOT_TEXT`                              |
| Free text / comments | `FPDF_ANNOT_FREETEXT`                          |
| Highlights           | `FPDF_ANNOT_HIGHLIGHT`                         |
| Underlines           | `FPDF_ANNOT_UNDERLINE`                         |
| Squiggly / strikeout | `FPDF_ANNOT_SQUIGGLY`, `FPDF_ANNOT_STRIKEOUT`  |
| Shapes               | `FPDF_ANNOT_SQUARE`, `FPDF_ANNOT_CIRCLE`, etc. |
| Popup notes          | `FPDF_ANNOT_POPUP`                             |

Key API surface: `FPDFAnnot_GetStringValue()` (contents, author),
`FPDFAnnot_GetAttachmentPoints()` (quadpoints for text markup),
`FPDFAnnot_GetRect()` / `FPDFAnnot_GetVertices()` / `FPDFAnnot_GetInkListPath()`
(shape geometry), `FPDFAnnot_GetColor()`.

**Important caveat:** Nearly the entire `fpdf_annot.h` API is marked
`// Experimental API.` Keep FFI bindings behind a Dart abstraction layer so any
signature changes are localised.
