# v0.01

## PDFium build infrastructure ✅ Complete

> Implemented in
> [`../plans/completed/plan_pdfium_build_infrastructure.md`](../plans/completed/plan_pdfium_build_infrastructure.md).

Bootstrap the PDFium native binary build on macOS arm64 using depot_tools and
Ninja, generate the initial Dart FFI bindings via `ffigen`, and validate that
the library loads correctly. This is the prerequisite that unblocks all
subsequent Dart implementation work.

- `make setup` — bootstraps the clean-room build workspace (depot_tools +
  gclient sync)
- `make build_pdfium_macos` — compiles and stages `libpdfium.dylib`
- `make ffi_bindings` — regenerates the Dart FFI bindings from public headers
- Smoke test confirms `FPDF_InitLibraryWithConfig` / `FPDF_DestroyLibrary`
  round-trip

## FFI bindings ✅ Complete

> Implemented as part of "../pdfium">
> [`../plans/completed/plan_pdfium_build_infrastructure.md`](../plans/completed/plan_pdfium_build_infrastructure.md).

Dart FFI bindings are generated with `ffigen` pointing at
`third_party/pdfium/public/` headers and committed to
`lib/src/generated/pdfium_bindings.dart` so developers without a C++ toolchain
can still build Dart code.

Bindings are added incrementally on an as-needed basis — only the headers
required by an implemented feature are in scope. `fpdfview.h` is included for
this release; additional headers are added as subsequent feature plans are
implemented.

## Metadata extraction (Info dictionary) ✅ Complete

> Implemented in
> [`../plans/completed/plan_metadata_extraction.md`](../plans/completed/plan_metadata_extraction.md).

Extracts the standard PDF Info dictionary fields using `FPDF_GetMetaText()`:
**Title, Author, Subject, Keywords, Creator, Producer, CreationDate, ModDate**.
Also exposes `FPDF_GetFileVersion()` and `FPDF_GetFileIdentifier()` via
`PdfDocument.getDocumentInfo()`.

Introduces `PdfDocument` as the top-level Dart abstraction with a
`PdfiumIsolate` singleton for thread-safe native FFI calls. A `pdfinfo` CLI tool
(`dart run bin/pdfinfo.dart`) is provided for manual testing against real-world
PDF files.

XMP metadata (the richer modern format) is deferred to v0.05 — see that
milestone for the chosen extraction approach and rationale.
