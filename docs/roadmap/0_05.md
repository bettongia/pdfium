# v0.05

## XMP metadata extraction

Planned.

> Implemented in [`../plans/plan_xmp_metadata_extraction.md`](../plans/plan_xmp_metadata_extraction.md).

PDFium does not expose the XMP stream through its public API. XMP metadata must
be read by parsing the `/Metadata` stream directly from the PDF file. XMP is
well-structured XML; the relevant namespaces are well-documented:

| Namespace | Prefix | Contents |
|-----------|--------|----------|
| Dublin Core | `dc:` | Title, creator, description, subject, rights |
| XMP Basic | `xmp:` | CreateDate, ModifyDate, CreatorTool |
| PDF | `pdf:` | Keywords, Producer |

**Chosen approach:** implement a Dart-native XMP parser that reads the
`/Metadata` stream directly from the PDF file, with a fallback to the Info
dictionary via `FPDF_GetMetaText()` for older PDFs that pre-date XMP. This
avoids a dependency on external libraries such as `exempi` or the Adobe XMP
Toolkit, keeps the implementation pure Dart, and is viable because XMP is
well-structured XML.

The dual-source strategy (XMP preferred, Info dictionary fallback) ensures
correct behaviour across both modern and legacy PDFs.
