# v0.01

## Cross-platform PDFium build pipeline

Planned.

> Implemented in
> [`../plans/plan_pdfium_build_pipeline.md`](../plans/plan_pdfium_build_pipeline.md).

**Prerequisite:** all core project goals (v0.01–v0.05) must be complete and the
Dart API must be stable before committing to cross-platform binary artifacts.

The macOS arm64 binary produced by the v0.01 build infrastructure is sufficient
for development and validation. This milestone establishes an orphan
`pdfium-build` branch within this repository that owns the full build matrix and
publishes versioned GitHub Release artifacts. `betto_pdfium` then fetches the
appropriate artifact at development time rather than building from source.

**Target platforms:**

| Platform       | Library form             | Status                                      |
| -------------- | ------------------------ | ------------------------------------------- |
| macOS arm64    | dylib                    | In scope                                    |
| macOS x86_64   | dylib                    | Dropped — Flutter dropping support          |
| Linux x86_64   | shared lib               | In scope                                    |
| Windows x86_64 | DLL                      | Dropped — no test environment, out of scope |
| iOS arm64      | static lib (xcframework) | In scope                                    |
| Android arm64  | shared lib               | In scope                                    |
| Android x86_64 | shared lib               | In scope                                    |
| Web (WASM)     | .wasm + .js              | In scope                                    |

A `make fetch_pdfium` target reads `PDFIUM_VERSION` (a full PDFium commit SHA)
and downloads the matching GitHub Release artifacts into
`third_party/pdfium_bin/`. Downloaded dylibs are ad-hoc signed after unpacking
to clear the `com.apple.quarantine` xattr that Gatekeeper would otherwise block.

## Mobile integration test app (iOS and Android)

Planned.

> Implemented in
> [`../plans/plan_mobile_integration_tests.md`](../plans/plan_mobile_integration_tests.md).

The desktop test suite (`dart test`) cannot exercise the iOS and Android
library-load paths — `DynamicLibrary.process()` (iOS static xcframework) and
bare-name `DynamicLibrary.open('libpdfium.so')` (Android) are platform-gated
branches that no host test can reach. This work creates a standalone Flutter
integration test app in `integration_test_app/` that runs the equivalent of the
desktop suite on a connected device or emulator.

**Scope:**

- A `fetch_mobile_binaries.sh` script downloads and verifies iOS and Android
  artifacts from the same GitHub Release used by the native-assets hook, using
  `version_pdfium.json` as the single source of truth for URLs and SHA-256
  digests.
- iOS: static xcframework linked via a local Swift Package Manager binary target;
  `DynamicLibrary.process()` after link.
- Android: `.so` placed in `jniLibs/`; loaded with `DynamicLibrary.open('libpdfium.so')`.
- Tests run manually (`flutter test integration_test/ -d <device-id>`); CI
  automation is out of scope.

**Status (2026-06-26):**

| Platform     | Artifact state       | Test status              |
| ------------ | -------------------- | ------------------------ |
| iOS arm64    | xcframework shipping | Pending (plan open)      |
| Android arm64 | Broken 5 KB stub    | Blocked on build fix     |
| Android x86_64 | Broken 5 KB stub  | Blocked on build fix     |

Android `.so` artifacts were broken (5 KB symbol-stripped stubs) due to a
missing `COMPONENT_BUILD` define; fix landed in `pdfium-build` 2026-06-26 and a
rebuilt release is pending. Android manifest entries and emulator verification
are deferred until a working artifact ships.
