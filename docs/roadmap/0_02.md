# v0.02

This roadmap will pick up work that was deferred from 0.01

## Web (WASM) binary

Web (WASM) was included in the v0.01 platform scope but the Emscripten build
was never implemented — the CI job in `build_pdfium.yml` is currently a
placeholder that does not produce a real artifact.

**Scope:**

- Wire up Emscripten in `pdfium-build/scripts/build_wasm.sh`, using the
  Emscripten version pinned by PDFium's own `DEPS` / `third_party/` rather
  than choosing one independently.
- Produce a `.wasm` + `.js` module pair matching the artifact layout defined
  in `docs/spec/01_binary_distribution.md`
  (`libpdfium-web.wasm`, `libpdfium-web.js`).
- Include both files in the checksums and publish them as part of the GitHub
  Release, so `make fetch_pdfium` can download and verify them alongside the
  native binaries.
- Replace the placeholder CI job with a real Emscripten build step on an
  `ubuntu-latest` runner.
- Implement `lib/src/document/_document_web.dart` — the WASM backend behind
  the `dart.library.js_interop` conditional import in `PdfDocument`.

**Prerequisite:** a working Android release (`libpdfium-android-*.so`) should
ship first so the release publishing path is fully exercised before adding WASM
artifacts.

## Windows x86_64 binary

Windows was dropped from v0.01 scope due to the absence of a local test
environment. It is deferred here for when a Windows runner or test environment
becomes available.

**Scope:**

- Add `pdfium-build/scripts/build_windows.sh` (or equivalent `.bat`/PowerShell
  script) and a corresponding `Makefile` target.
- Configure a GitHub Actions `windows-latest` runner job in `build_pdfium.yml`
  to build and smoke-test a `libpdfium-windows-x86_64.dll`.
- Add the DLL to the artifact layout in `docs/spec/01_binary_distribution.md`
  and to the checksums/release step.
- Update `make fetch_pdfium` to download and verify the Windows DLL.
- Add a `lib/src/document/_document_native.dart` library-load path for
  `Platform.isWindows` (bare `DynamicLibrary.open('libpdfium.dll')`).

**Constraint:** a functioning Windows build environment (local or CI) is a hard
prerequisite before this item can be planned in detail.
